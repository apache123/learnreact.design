React

Now meet our model agent, React:

With React, we no longer directly talk to Domo. React acts as a middleman between a developer and Domo. She smoothes out the communication and streamlines the process of portrait creation.

React has a few tricks to address the issues of jQuery and other tools. Here are her three main tricks:

  • Reactive UI
  • Virtual DOM
  • Components

Reactive UI

To update the DOM with jQuery, you‚Äôd have to specify what element to change at the right time, in the right order. This is equivalent to describing to Domo how to position his head, arms and legs step by step, for each and every portrait.

Heck, this sounds tedious and error-prone! Why can‚Äôt we just tell Domo what we want instead of how to pose2?

More importantly, you can leave placeholders in your request to represent different variations of a same pose.

This trick is how React got its name. The UI built with React is reactive. As a developer, you just need to write down what you want and React figures out how to do it. When the data change, your UI changes accordingly. You don‚Äôt need to worry about updating the DOM, React does it for you automatically. The idea of reactive UI greatly simplifies UI development.

Virtual DOM

Another issue of jQuery is about the speed. As customers of the studio, we are very demanding. We want the portraits to come out as quickly as possible. We don‚Äôt like to wait.

However, Domo and Browsera are slow. Not really as slow as a sloth, but it takes time for Domo to change clothes and poses, and it takes time for Browsera to paint.

React pulls another trick to fix that. She‚Äôs very fast at drawing quick sketches. Almost immediately after you tell her your requirement, she‚Äôs done with a sketch and ready to take the next order. Now no more wait! You can keep telling React about the portraits you want, nonstop. React keeps record of everything with sketches and show it to Domo at the right time.

More importantly, React is smart. He can sort through the sketches, remove any duplication and make sure that Domo and Browsera do as little work as possible.

These sketches are called ‚ÄúVirtual DOM‚Äù. Virtual DOM is much faster to manipulate than DOM. Developers work with Virtual DOM most of the time instead of directly managing DOM. React handles the dirty work of managing the slow DOM.

Components

The third trick of React is the concept of components.

Components should be straightforward to understand because it‚Äôs the way how our real world works. Our cars, houses and even our bodies are all made up of identifiable components that are individual, functional units. These components are made up of smaller components which consist of even smaller components, all the way down to atoms.

Back to our studio ‚ÄúWeb Browser‚Äù, you describe your requirement for portraits as components, React translates it into what Domo understands. This saves you a lot time because you don‚Äôt have to repeat common parts in your order.

Another cool thing about components is that, if you change a component, everything that includes this component will be updated automatically.
